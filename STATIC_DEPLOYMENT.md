# Static deployment workflow

Use the ASP.NET Core app locally as the content management tool. The public production site is generated by snapshotting the real public Razor pages as static files.

## Local editing

Run the local app:

```powershell
dotnet run --project C:\Dev\Playground\Codex\BookBoutique\BookBoutique.csproj --urls http://0.0.0.0:5099
```

Open the admin area:

```text
http://localhost:5099/Admin
```

Use the admin pages to edit bonus chapters, podcasts, and the image gallery. Bonus chapters use uploaded PDF files. These changes update `App_Data` and `wwwroot/media`.

## Generate the static site

After editing content, run:

```powershell
dotnet run --project C:\Dev\Playground\Codex\BookBoutique\BookBoutique.csproj -- export-static
```

The deployable site is written to:

```text
C:\Dev\Playground\Codex\BookBoutique\dist
```

Only public pages are exported. Admin pages are not included.

During export, the app starts locally on a temporary port, requests the public pages, rewrites public links to static paths, and saves those responses into `dist`.

## Deploy with GitHub Actions

The production site is deployed as a Cloudflare Worker with static assets named `bookstore`.

The GitHub Actions workflow in `.github/workflows/deploy.yml` runs on every push to `main`:

1. restore and build the .NET project
2. run `export-static`
3. deploy `dist` to Cloudflare with Wrangler

Configure these GitHub repository secrets:

```text
CLOUDFLARE_API_TOKEN=Cloudflare API token with Workers Scripts edit permission
CLOUDFLARE_ACCOUNT_ID=Cloudflare account ID
```

The Worker deployment config is in `wrangler.jsonc`.

## Order API Worker

The order page is static, but order submission uses a separate Cloudflare Worker:

```text
api/ein-hamelech-orders.js
```

The current Cloudflare route is:

```text
ein-hamelech.shakedshira.com/api/order
```

Configure these Cloudflare Worker variables/secrets:

```text
RESEND_API_KEY=your Resend API key
ORDER_EMAIL_TO=the mailbox that should receive new orders
ORDER_EMAIL_FROM=verified sender address in Resend
```

The static-site workflow deploys only the `bookstore` Worker. Deploy `api/ein-hamelech-orders.js` separately when the order email Worker changes.

The exported site includes:

- public HTML pages
- CSS and JavaScript
- the hero video
- uploaded gallery images
- uploaded podcast audio files
- uploaded bonus chapter PDF files
- one static page per published bonus chapter under `dist\bonus`
- one QR SVG per published bonus chapter under `dist\qr`

Bonus chapter URLs are based on the chapter ID, so editing the title, summary, or uploaded PDF keeps the same QR URL. Deleting a chapter and creating a new one creates a new URL.
